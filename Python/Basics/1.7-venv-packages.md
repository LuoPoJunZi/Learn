# 1.7 虚拟环境与第三方库

Python 强大的原因之一，是有大量第三方库。但项目一多，如果所有库都装在全局环境里，很容易乱。虚拟环境就是为了解决这个问题。

## pip 是什么

`pip` 是 Python 的包管理工具，用来安装第三方库。

查看版本：

```bash
pip --version
```

安装库：

```bash
pip install requests
```

卸载库：

```bash
pip uninstall requests
```

查看已安装库：

```bash
pip list
```

## 虚拟环境是什么

虚拟环境可以为每个项目创建独立的依赖环境。

例如：

- 项目 A 使用 `requests==2.28`
- 项目 B 使用 `requests==2.31`

如果没有虚拟环境，这些版本可能互相影响。

## 创建虚拟环境

在项目目录下执行：

```bash
python -m venv .venv
```

会生成 `.venv` 文件夹。

## 激活虚拟环境

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
```

Windows CMD：

```cmd
.venv\Scripts\activate.bat
```

macOS / Linux：

```bash
source .venv/bin/activate
```

激活后，终端前面通常会出现：

```text
(.venv)
```

## 安装依赖

激活虚拟环境后：

```bash
pip install requests pandas openpyxl
```

## 导出依赖

```bash
pip freeze > requirements.txt
```

`requirements.txt` 会记录当前环境安装了哪些库。

## 安装 requirements

别人拿到项目后，可以执行：

```bash
pip install -r requirements.txt
```

## 退出虚拟环境

```bash
deactivate
```

## .gitignore 建议

虚拟环境不要上传到 GitHub。

`.gitignore` 中建议加入：

```gitignore
.venv/
__pycache__/
*.pyc
```

## 常见问题

### pip 不是内部或外部命令

可能是 Python 没有加入 PATH。重新安装 Python，并勾选 `Add Python to PATH`。

### PowerShell 不允许激活脚本

可能是执行策略限制。可以临时使用：

```powershell
powershell -ExecutionPolicy Bypass
```

然后在新开的 PowerShell 中激活虚拟环境。

### 安装库很慢

可能是网络问题。可以换网络环境，或配置合适的软件源。不要随便复制不明来源的安装命令。
