# 文档整理规范

这个仓库会持续加入学习笔记、教程和脚本。为了后续内容不变散，新增或整理文档时建议遵守下面的规范。

## 目录入口

每个一级目录建议有一个 `README.md`，说明：

- 这个目录放什么内容
- 推荐阅读顺序
- 子目录或文件索引
- 运行或使用前的注意事项

大型专题可以继续在子目录添加 README，例如 `Matlab/Neural Network/README.md`。

## 教程结构

一篇教程建议包含：

```markdown
# 标题

这篇文章解决什么问题，适合谁看。

## 背景

为什么需要这个知识点？

## 操作步骤

1. 第一步
2. 第二步
3. 第三步

## 常见问题

## 参考来源
```

## 脚本文档结构

脚本目录的 README 建议包含：

- 脚本功能
- 运行环境
- 依赖安装
- 使用示例
- 输入和输出
- 风险提醒

涉及账号、Token、密码、API Key 的脚本必须写安全提醒。

## 链接规范

- 仓库内链接优先使用相对路径。
- 路径包含空格时，用尖括号包住，例如 `[Neural Network](<Neural Network/README.md>)`。
- 外部链接尽量标明来源用途。
- 删除或移动文件后，要同步检查 README 链接。

## 代码块规范

代码块尽量标注语言：

````markdown
```bash
git status
```
````

常见语言：

- `bash`
- `powershell`
- `python`
- `matlab`
- `markdown`
- `javascript`
- `html`
- `css`
- `json`

## 风险提醒

遇到下面内容时，需要明确提醒：

- 删除、重装、格式化、覆盖文件
- 远程脚本一键执行
- SSH、防火墙、VPS 网络配置
- API Key、Token、密码
- 爬虫、批量请求、批量下载
- 批量重命名、移动、删除文件

## 来源标注

如果内容参考了博客、课程、开源项目，建议在文末保留：

```markdown
## 参考来源

- 来源名称：链接
```

如果是经过整理、改写或补充，也可以说明“本文基于某资料整理，并补充了使用说明”。
