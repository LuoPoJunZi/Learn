# 新手检查清单合集

这份文档把仓库里常用的检查动作集中到一起。遇到不确定时，先按清单过一遍，很多问题会自己浮出来。

## 写文档前

- 我知道这篇文档给谁看。
- 我知道读者读完要做什么。
- 我准备好了推荐阅读顺序。
- 我知道需要链接到哪些旧文档。
- 我会在最后写下一步入口。

## 提交到 GitHub 前

- 已运行 `git status`。
- 暂存文件都是本次需要提交的内容。
- 没有密码、Token、私钥、临时文件。
- 提交信息能说明本次修改目的。
- 如果改了文档，已经运行文档检查脚本。

## 修改 Linux 服务器前

- 已确认当前用户和当前路径。
- 已备份重要配置。
- 已知道命令会读、写、删还是重启服务。
- 修改 SSH、防火墙、服务配置前有回滚方式。
- 远程脚本已经先读过内容或确认来源。

## 运行 Python 脚本前

- 已确认 Python 版本。
- 已激活正确虚拟环境。
- 已安装依赖。
- 已确认输入路径和输出路径。
- 批量处理文件前先在测试目录运行。

## 运行 Matlab 案例前

- 已确认当前工作目录。
- 已找到主脚本。
- 已确认数据文件存在。
- 已确认需要的工具箱。
- 已知道主要输出是什么。

## 运行 Mathematica Notebook 前

- 已清理可能影响结果的变量。
- 已确认函数调用使用 `[]`。
- 已区分赋值 `=` 和方程 `==`。
- 绘图变量范围已写清楚。
- 结果旁边有文字解释。

## 新增仓库内容前

- 新文件是否有入口链接。
- 对应目录 README 是否更新。
- 根 README 是否需要更新。
- `docs/LEARNING_PATH.md` 是否需要更新。
- `CHANGELOG.md` 是否需要记录。
