# 后续深入路线

如果你已经读完入门导读，却不知道下一步该深入哪里，可以用这份路线做选择。它按目标分类，而不是按目录分类。

## 路线一：文档与知识库维护

适合你如果：

- 想把学习资料整理清楚。
- 想写 README、教程、博客。
- 想维护这个仓库的结构。

推荐顺序：

1. [Markdown 写作进阶指南](../Markdown/WRITING_GUIDE.md)
2. [README 审查清单](../Markdown/README_REVIEW_CHECKLIST.md)
3. [Markdown 常见问题](../Markdown/FAQ.md)
4. [文档整理规范](STYLE_GUIDE.md)
5. [文档维护者指南](../scripts/MAINTAINER_GUIDE.md)

阶段目标：能新增一篇教程，并正确更新 README、学习路线和更新日志。

## 路线二：GitHub 协作能力

适合你如果：

- 想参与团队项目。
- 想给开源项目贡献内容。
- 想理解 PR 和 Review。

推荐顺序：

1. [GitHub 真实场景指南](../Github/SCENARIOS.md)
2. [Pull Request Review 入门指南](../Github/REVIEW_GUIDE.md)
3. [GitHub 学习导读](../Github/LEARNING_GUIDE.md)
4. [Pull Request 教程](../Github/docs/part3-teamwork/3.2-pull-request.md)
5. [冲突解决](../Github/docs/part3-teamwork/3.3-merge-conflict.md)

阶段目标：能从分支发起 PR，并根据 Review 修改。

## 路线三：VPS 与 Linux 运维

适合你如果：

- 想管理自己的服务器。
- 想排查服务、端口、日志、SSH。
- 经常使用一键脚本但想理解风险。

推荐顺序：

1. [Linux VPS 日常运维导读](../Linux/OPERATIONS_GUIDE.md)
2. [Linux VPS 基础安全清单](../Linux/SECURITY_BASELINE.md)
3. [Linux 学习导读](../Linux/LEARNING_GUIDE.md)
4. [VPS 操作安全清单](../Linux/docs/part4-vps-tools/4.2-vps-safety-checklist.md)
5. [SSH 安全加固](../Linux/docs/part3-network-ssh/3.2-ssh-security.md)

阶段目标：能判断服务访问失败时应该先看状态、日志、端口还是防火墙。

- 想处理文件、表格、图片、PDF。
- 想把现有脚本改成自己的版本。

推荐顺序：

1. [Python 小项目学习路线](../Python/PROJECT_ROADMAP.md)
2. [Python 环境与依赖管理指南](../Python/ENVIRONMENT_GUIDE.md)
3. [Python 学习导读](../Python/LEARNING_GUIDE.md)
4. [Python 常见问题](../Python/FAQ.md)
5. [Auto_scripts 脚本索引](../Python/Auto_scripts/README.md)

阶段目标：能选择一个脚本，读懂输入输出，并改成适合自己目录结构的工具。

## 路线五：算法与建模

适合你如果：

- 想学习优化算法、神经网络或数学建模。
- 想对照 Matlab、Python、Mathematica 的能力。

推荐顺序：

1. [Matlab 案例阅读指南](../Matlab/CASE_READING_GUIDE.md)
2. [Matlab 数据与绘图入门指南](../Matlab/DATA_PLOT_GUIDE.md)
3. [Matlab 学习导读](../Matlab/LEARNING_GUIDE.md)
4. [Python 多目标优化示例](<../Python/Multi-Objective Optimization/README.md>)
5. [Mathematica 专题学习指南](../Mathematica/TOPIC_GUIDE.md)
6. [Mathematica Notebook 写作指南](../Mathematica/NOTEBOOK_GUIDE.md)

阶段目标：能跑通一个算法案例，并解释它的输入、核心参数和输出结果。

## 如何判断自己该换路线

如果你连续三次遇到同一类问题，就说明该补对应路线：

- 文档写不清：补 Markdown。
- 提交和分支混乱：补 GitHub。
- 服务器命令看不懂：补 Linux。
- 脚本跑不起来：补 Python 环境。
- 算法案例看不懂：补 Matlab 或 Mathematica 基础。

学习路线不是固定轨道，是地图。你可以随时根据当前问题切换。
