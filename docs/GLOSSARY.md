# 新手术语表

这份术语表解释本仓库中反复出现的关键词。它不是严格百科，而是给新手快速建立直觉。

## GitHub 与 Git

| 术语 | 解释 |
| :--- | :--- |
| Git | 本地版本控制工具，用来记录文件变化。 |
| GitHub | 远程代码托管和协作平台。 |
| Repository / 仓库 | 保存项目文件和历史记录的地方。 |
| Commit / 提交 | 一次明确的版本保存。 |
| Branch / 分支 | 与主线隔离的一条开发线。 |
| Pull Request / PR | 请求把一个分支的修改合并到另一个分支。 |
| Fork | 把别人的仓库复制到自己的账号下，常用于开源贡献。 |
| Conflict / 冲突 | 多个修改影响同一位置，需要人工选择最终内容。 |

## Linux

| 术语 | 解释 |
| :--- | :--- |
| Shell | 接收命令并执行的程序。 |
| Terminal / 终端 | 输入命令的窗口。 |
| Path / 路径 | 文件或目录的位置。 |
| Permission / 权限 | 控制谁可以读、写、执行文件。 |
| Process / 进程 | 正在运行的程序实例。 |
| Service / 服务 | 由系统管理、常驻后台的程序。 |
| SSH | 远程登录服务器的安全协议。 |
| Port / 端口 | 网络服务监听的位置编号。 |

## Markdown

| 术语 | 解释 |
| :--- | :--- |
| Markdown | 用纯文本写格式化文档的标记语言。 |
| README | 项目首页说明文档。 |
| Code Fence | 三个反引号包住的代码块。 |
| Mermaid | 用文本画流程图、时序图等图表的工具。 |
| GFM | GitHub Flavored Markdown，GitHub 使用的 Markdown 扩展。 |

## Python

| 术语 | 解释 |
| :--- | :--- |
| Interpreter / 解释器 | 执行 Python 代码的程序。 |
| Module / 模块 | 一个可导入的 Python 文件或包。 |
| Package / 包 | 一组模块的集合。 |
| venv / 虚拟环境 | 为单个项目隔离依赖的 Python 环境。 |
| pip | Python 常用包安装工具。 |
| Exception / 异常 | 程序运行时出现的错误或特殊情况。 |

## Matlab 与 Mathematica

| 术语 | 解释 |
| :--- | :--- |
| Matrix / 矩阵 | 按行列组织的数据结构。 |
| Toolbox / 工具箱 | Matlab 中扩展功能的组件。 |
| Script / 脚本 | 按顺序执行的一组命令。 |
| Function / 函数 | 有输入和输出的可复用代码。 |
| Symbolic Computation / 符号计算 | 对公式本身做推导和化简。 |
| Rule / 规则 | Mathematica 中常见的替换形式，如 `x -> 1`。 |
| Notebook | Mathematica 的交互式文档和代码环境。 |

## 文档维护

| 术语 | 解释 |
| :--- | :--- |
| Link Check | 检查文档链接是否能找到目标。 |
| Changelog | 记录重要改动的更新日志。 |
| Style Guide | 统一文档写法和结构的规范。 |
| Scope | 一次修改包含和不包含的范围。 |

## 使用建议

第一次遇到陌生词，不需要立刻背下来。先回到对应目录的学习导读，看它出现在什么场景里，再结合实际操作理解。
