# 内容阅读指南

这个仓库内容比较多，新手不需要从头到尾一次读完。更好的方式是先按目标选择路线，再逐步补基础。

## 如果你完全不知道从哪里开始

按这个顺序：

1. [知识地图总览](KNOWLEDGE_MAP.md)
2. [30 天学习计划](30_DAY_PLAN.md)
3. [新手学习路线](LEARNING_PATH.md)
4. [新手术语表](GLOSSARY.md)
5. [新手常见问题排查手册](TROUBLESHOOTING.md)

先建立地图，再进入具体目录。

## 如果你想写好文档

推荐路线：

1. [Markdown 学习导读](../Markdown/LEARNING_GUIDE.md)
2. [Markdown 常见问题](../Markdown/FAQ.md)
3. [README 模板](../Markdown/examples/readme-template.md)
4. [文档整理规范](STYLE_GUIDE.md)

目标是能写出别人看得懂、自己以后也能维护的 README。

## 如果你想管理 GitHub 仓库

推荐路线：

1. [GitHub 学习导读](../Github/LEARNING_GUIDE.md)
2. [GitHub 常见问题](../Github/FAQ.md)
3. [GitHub 新手入门教程](../Github/README.md)

先理解工作区、暂存区、本地仓库、远程仓库，再学习分支和 PR。

## 如果你想操作服务器

推荐路线：

1. [Linux 学习导读](../Linux/LEARNING_GUIDE.md)
2. [Linux 常见问题](../Linux/FAQ.md)
3. [Linux 新手入门与 VPS 实用手册](../Linux/README.md)

重点掌握路径、权限、服务、日志、SSH。不要一开始就追求部署复杂服务。

## 如果你想写自动化脚本

推荐路线：

1. [Python 学习导读](../Python/LEARNING_GUIDE.md)
2. [Python 常见问题](../Python/FAQ.md)
3. [Python 基础教程](../Python/Basics/README.md)
4. [Auto_scripts 脚本索引](../Python/Auto_scripts/README.md)

先学会运行和改脚本，再逐步学习函数、模块、依赖和命令行参数。

## 如果你想做计算、算法和建模

推荐路线：

1. [Matlab 学习导读](../Matlab/LEARNING_GUIDE.md)
2. [Matlab 常见问题](../Matlab/FAQ.md)
3. [Mathematica 学习导读](../Mathematica/LEARNING_GUIDE.md)
4. [Mathematica 常见问题](../Mathematica/FAQ.md)

如果你偏工程实验和算法代码，先看 Matlab。如果你偏公式推导和符号计算，先看 Mathematica。

## 阅读时的三个习惯

1. 读一篇文档，就写 3 行总结。
2. 看到命令，先判断它会读、写、删还是上传文件。
3. 遇到报错，先复制完整报错，再查 FAQ 和排查手册。

## 下一步

如果你已经选好方向，就从对应目录的 `LEARNING_GUIDE.md` 开始。这个仓库的内容不是用来一口气读完的，而是用来在你遇到具体问题时，有地方可查、有路线可走。
