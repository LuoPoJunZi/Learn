# 学习记录模板

这份模板用于记录你每次学习、调试、运行脚本或整理文档的过程。它的重点不是写得漂亮，而是让你以后能看懂自己当时做了什么、遇到了什么问题、最后怎么解决。

## 为什么要记录

新手学习时最容易发生三件事：

- 今天解决了问题，明天忘了怎么解决。
- 看懂了教程，但不知道自己有没有真的会做。
- 报错时到处搜索，解决后没有留下线索。

学习记录可以帮你把“临时会了”变成“以后还能复现”。

## 适合不同场景的写法

### 学教程时

建议记录：

- 这篇教程讲什么。
- 哪些概念第一次见。
- 哪些命令自己跑过。
- 哪一步容易出错。
- 学完后能做什么。

示例：

```markdown
## 我真正理解了什么

- `git add` 不是提交，只是把文件放进暂存区。
- `git commit` 是保存到本地仓库。
- `git push` 才是上传到 GitHub。
```

### 跑脚本时

建议记录：

- 脚本路径。
- Python 或 Matlab 版本。
- 安装了哪些依赖。
- 输入文件是什么。
- 输出文件在哪里。
- 是否有危险操作。

示例：

```markdown
## 运行环境

- Python：3.11
- 脚本：Python/Auto_scripts/rename_files
- 输入目录：D:\test-files
- 结果：成功批量重命名 10 个文件
```

### 排查报错时

建议记录：

- 完整报错。
- 你运行的命令。
- 当前目录。
- 你尝试过的方法。
- 最终有效的方法。

可以搭配阅读：

- [新手常见问题排查手册](TROUBLESHOOTING.md)

示例：

````markdown
## 遇到的问题

### ModuleNotFoundError: No module named 'requests'

现象：

运行脚本时报错，提示找不到 `requests`。

原因：

当前虚拟环境没有安装 `requests`。

解决：

```bash
pip install requests
```
````

### 整理文档时

建议记录：

- 修改了哪些文件。
- 为什么要这样整理。
- 是否保留了原始内容入口。
- 是否跑过文档检查脚本。

示例：

```markdown
## 最后结果

- 新增 README 入口。
- 原来的文档没有删除，只是增加了索引。
- 已运行 `scripts/check-docs.ps1`，检查通过。
```

## 好记录的标准

一篇好的学习记录通常满足：

- 以后自己能看懂。
- 能复现关键步骤。
- 报错和解决方法写清楚。
- 命令和代码没有只截一半。
- 链接能打开。
- 危险操作有提醒。

不用追求文采。学习记录最重要的价值是“可复现”和“可回头看”。
