# Mathematica Notebook 写作指南

Notebook 不只是代码运行窗口，它更像一份“可以执行的数学笔记”。写得好的 Notebook 能同时保存问题、推导、代码、图像和结论。

## 一个好的 Notebook 应该包含什么

建议结构：

```text
问题背景
-> 符号定义
-> 推导或计算
-> 数值验证
-> 绘图
-> 结果解释
```

不要只堆代码。以后回看时，文字说明和结果解释比代码本身更重要。

## 标题和分段

每个专题建议有清晰标题：

```text
1. 问题定义
2. 符号推导
3. 数值计算
4. 可视化
5. 结论
```

如果一个 Notebook 同时包含多个问题，建议拆成多个文件或多个清晰章节。

## 变量命名

建议：

- 符号变量用简单字母，如 `x`、`t`。
- 中间结果用有意义的名字。
- 重要计算前先 `Clear`。

示例：

```wolfram
Clear[x, t]
expr = Sin[x]^2 + Cos[x]^2;
Simplify[expr]
```

如果变量曾经被赋值，后续符号计算可能会出现意外结果。

## 精确计算和数值计算

Notebook 中建议先保留精确表达式，再给数值近似：

```wolfram
exact = 1/3;
numeric = N[exact]
```

这样读者能同时看到数学形式和近似结果。

## 绘图前后写解释

绘图前说明你想观察什么：

```wolfram
Plot[Sin[x], {x, 0, 2 Pi}]
```

绘图后写一句解释：

```text
图中可以看到函数在 0 到 2 Pi 内完成一个周期。
```

图不是结论本身，解释才是。

## 保存和导出

Notebook 适合交互阅读。如果要分享结果，可以考虑导出为 PDF 或截图。

导出前检查：

- 代码是否按顺序运行。
- 变量是否清理干净。
- 图形是否能重新生成。
- 文字解释是否完整。

## 常见问题

| 问题 | 建议 |
| :--- | :--- |
| 后面计算被前面变量影响 | 使用 `Clear` 清理变量。 |
| 图画不出来 | 检查变量范围和变量是否已有值。 |
| 结果不是小数 | 使用 `N` 转数值。 |
| Notebook 太长 | 按专题拆分。 |
| 只有代码没有解释 | 每段代码前后补一句说明。 |
