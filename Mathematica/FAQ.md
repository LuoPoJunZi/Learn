# Mathematica 常见问题

Mathematica / Wolfram Language 的语法和很多语言不一样。新手最容易卡在括号、赋值、规则替换、精确计算和绘图范围上。

## `[]`、`{}`、`()` 分别做什么？

常见用途：

- `[]`：函数调用，例如 `Sin[x]`。
- `{}`：列表，例如 `{1, 2, 3}`。
- `()`：改变运算优先级，例如 `(x + 1)^2`。

如果你来自 Python 或 Matlab，最需要适应的是函数调用使用方括号。

## `=` 和 `==` 有什么区别？

`=` 是赋值：

```wolfram
x = 3
```

`==` 是方程或判断：

```wolfram
x^2 == 9
```

求解方程时通常使用 `==`，不要写成 `=`。

## `->` 和 `/.` 是什么？

`->` 表示规则，`/.` 表示应用替换规则。

```wolfram
x^2 + 1 /. x -> 2
```

这表示把表达式中的 `x` 替换成 `2`。

## 为什么结果是分数，不是小数？

Mathematica 默认会尽量保持精确计算。

```wolfram
1/3
```

如果想得到近似小数：

```wolfram
N[1/3]
```

或者一开始使用小数：

```wolfram
1.0/3
```

## 为什么 `Plot` 没有画出图？

常见原因：

- 没有给变量范围。
- 函数在范围内没有实数值。
- 变量写错。

基本写法：

```wolfram
Plot[Sin[x], {x, 0, 2 Pi}]
```

`{x, 0, 2 Pi}` 表示变量和绘图范围。

## 为什么变量值一直影响后面的计算？

你可能已经给变量赋过值。可以清除：

```wolfram
Clear[x]
```

做符号计算前，如果结果奇怪，先确认变量是否被赋值。

## Mathematica 适合替代 Python 或 Matlab 吗？

它们各有强项。

- Mathematica：符号计算、公式推导、快速可视化。
- Matlab：矩阵计算、工程算法、工具箱生态。
- Python：通用编程、自动化、数据处理、开源生态。

实际学习中，可以把同一个问题用不同工具实现一遍，对照理解。

## Notebook 里应该怎么整理内容？

建议按“文字说明 + 代码 + 结果解释”组织。不要只堆代码单元。

一个好的 Notebook 应该像一篇可执行的学习笔记。
