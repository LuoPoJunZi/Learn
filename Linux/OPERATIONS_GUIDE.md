# Linux VPS 日常运维导读

这篇文档面向刚开始使用 VPS 的新手，重点讲日常维护时的判断顺序。它不是命令大全，而是一张“服务器出问题时该先看哪里”的地图。

## 运维前的基本原则

- 先查看状态，再修改配置。
- 先备份重要文件，再执行删除或覆盖。
- 先读日志，再重装服务。
- 先确认路径，再使用递归命令。
- 不把一键脚本当作黑盒，至少先看一眼它会做什么。

## VPS 初始化顺序

新服务器建议按这个顺序做：

1. 记录系统版本和公网 IP。
2. 更新系统软件包。
3. 创建普通用户。
4. 配置 SSH 密钥登录。
5. 检查防火墙和开放端口。
6. 安装必要工具，例如 `curl`、`wget`、`vim`、`git`。
7. 记录已安装服务和配置文件位置。

查看系统信息：

```bash
cat /etc/os-release
uname -a
```

## 服务排查顺序

当一个服务无法访问时，不要马上重装。按下面顺序排查：

```text
服务是否运行
-> 日志是否报错
-> 配置是否正确
-> 端口是否监听
-> 防火墙是否放行
-> 域名或反向代理是否正确
```

常用命令：

```bash
systemctl status nginx
journalctl -u nginx -n 100
ss -tulnp
```

如果日志里出现配置文件路径，优先打开对应文件查看。

## 磁盘空间不足

磁盘满了会导致服务启动失败、日志无法写入、数据库异常。

先看总体空间：

```bash
df -h
```

再看目录大小：

```bash
du -sh /var/log
du -sh /home/*
```

清理前先确认文件用途。日志可以按服务规则轮转或清理，数据库文件和项目数据不要随便删除。

## 内存和进程异常

查看内存：

```bash
free -h
```

查看进程：

```bash
ps aux --sort=-%mem | head
```

不要看到某个进程占用高就立刻 `kill -9`。先确认它是什么服务，是否正在执行正常任务。

## 网络连不通

排查顺序：

1. 本机能否解析域名。
2. 服务器端口是否监听。
3. 防火墙是否放行。
4. 服务是否绑定到正确地址。
5. 云厂商安全组是否开放端口。

常用命令：

```bash
ping example.com
curl -I https://example.com
ss -tulnp
```

如果 `curl localhost:端口` 能通，但外网不通，通常要查防火墙、安全组或反向代理。

## 修改配置文件的安全流程

修改系统配置前建议：

```bash
cp config.conf config.conf.bak
```

修改后先测试配置，再重启服务。例如 Nginx：

```bash
nginx -t
systemctl reload nginx
```

如果只是改了配置，优先 `reload`。只有服务状态异常或必须重启时再 `restart`。

## 一键脚本使用提醒

一键脚本很方便，但也有风险。运行前至少确认：

- 脚本来源是否可信。
- 是否需要 root 权限。
- 是否会修改 SSH、防火墙、系统源。
- 是否会安装后台服务。
- 是否有卸载或回滚方式。

远程脚本不要盲目执行。能先打开看看，就先打开看看。

## 日常维护记录

建议给每台 VPS 建一个简单记录：

```text
系统版本：
用途：
开放端口：
已安装服务：
重要配置文件：
备份位置：
最近修改：
```

记录不是形式主义。出问题时，它能帮你少猜很多东西。
