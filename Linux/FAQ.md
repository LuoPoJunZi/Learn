# Linux 常见问题

这份 FAQ 主要面向刚接触 Linux、VPS、SSH 的新手。它不追求覆盖所有命令，而是帮你理解常见报错背后的原因。

## 为什么我输入命令后提示 `command not found`？

常见原因：

- 命令拼错了。
- 软件没有安装。
- 当前系统发行版不包含这个命令。
- 命令所在目录不在 `PATH` 中。

可以先检查：

```bash
which 命令名
命令名 --version
```

如果是软件没安装，再根据系统使用 `apt`、`dnf`、`yum` 等包管理器安装。

## 为什么提示 `Permission denied`？

这通常和权限有关，不一定是系统坏了。

常见情况：

- 当前用户没有读取或写入权限。
- 文件没有执行权限。
- SSH 密钥权限过宽。
- 目标目录属于其他用户。

先查看：

```bash
ls -lah
whoami
```

不要第一反应就 `chmod 777`。这会把权限放得过大，可能带来安全风险。

## `sudo` 是什么？

`sudo` 表示以更高权限执行命令。它适合安装软件、修改系统配置、管理服务。

使用前先问自己：

- 这个命令会不会删除或覆盖文件？
- 目标路径是不是系统目录？
- 我是否知道命令每个参数的含义？

新手复制带 `sudo` 的命令前，最好先拆开理解。

## 为什么服务启动失败？

不要急着重装。先看状态和日志：

```bash
systemctl status 服务名
journalctl -u 服务名 -n 100
```

常见原因：

- 配置文件语法错误。
- 端口被占用。
- 文件路径不存在。
- 权限不对。
- 依赖服务没有启动。

日志通常会直接给出关键线索。

## 端口被占用是什么意思？

一个端口同一时间通常只能被一个服务监听。如果新服务想用的端口已经被占用，就会启动失败。

可以检查：

```bash
ss -tulnp
```

看到占用进程后，不要直接杀掉，先确认它是什么服务、是否可以停止。

## SSH 登录失败怎么办？

排查顺序：

1. 确认服务器 IP 和端口。
2. 确认用户名。
3. 确认密钥或密码是否正确。
4. 检查本地网络是否能连接服务器。
5. 如果还能通过控制台进入服务器，再看 SSH 日志。

相关内容可以看 [SSH 安全加固](docs/part3-network-ssh/3.2-ssh-security.md)。

## `rm -rf` 为什么危险？

`rm -rf` 会递归删除文件，而且不会进入回收站。路径写错时，后果可能很严重。

执行前建议：

```bash
pwd
ls -lah 目标路径
```

确认路径之后再执行删除。重要服务器上，先备份再删。

## 为什么教程里的命令和我的系统不一样？

Linux 有不同发行版，例如 Ubuntu、Debian、CentOS、Fedora、Arch。它们的包管理器、默认服务名、配置路径可能不同。

遇到差异时先确认：

```bash
cat /etc/os-release
uname -a
```

然后再查对应发行版的文档。
