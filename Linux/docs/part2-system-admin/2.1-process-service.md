# 2.1 进程、服务与后台任务

进程是正在运行的程序。服务是长期在后台运行的进程，例如 SSH、Nginx、Docker。

## 查看进程

```bash
ps aux
ps aux | grep nginx
top
htop
```

- `ps aux`: 查看当前进程列表。
- `grep`: 过滤关键词。
- `top`: 实时查看资源占用。
- `htop`: 更友好的进程查看工具，可能需要额外安装。

## 结束进程

```bash
kill PID
killall name
pkill name
```

- `kill PID`: 按进程 ID 结束。
- `killall`: 按进程名结束。
- `pkill`: 按名称匹配结束。

结束系统关键服务前要谨慎。比如远程服务器上不要随手 `killall sshd`，否则可能把自己的 SSH 连接断掉。

## 后台任务

```bash
command &
jobs
fg
bg
nohup command &
```

- `&`: 让命令在后台运行。
- `jobs`: 查看当前终端的后台任务。
- `fg`: 切回前台。
- `bg`: 继续后台运行。
- `nohup`: 终端关闭后命令仍继续运行。

## systemctl 管理服务

```bash
sudo systemctl status ssh
sudo systemctl start ssh
sudo systemctl stop ssh
sudo systemctl restart ssh
sudo systemctl enable ssh
sudo systemctl disable ssh
```

- `status`: 查看状态。
- `start`: 启动。
- `stop`: 停止。
- `restart`: 重启。
- `enable`: 开机自启。
- `disable`: 取消开机自启。

不同发行版中 SSH 服务名可能是 `ssh` 或 `sshd`：

```bash
sudo systemctl status ssh
sudo systemctl status sshd
```

## 查看服务日志

```bash
journalctl -u ssh
journalctl -u ssh -f
journalctl -u ssh --since "1 hour ago"
```

`-f` 表示实时跟踪日志，适合排查服务启动失败或登录问题。
