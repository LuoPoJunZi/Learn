# 1.4 权限与所有者

Linux 权限决定谁能读、写、执行文件。理解权限后，你才能看懂 `ls -l` 输出，也能避免把权限改得过于危险。

## 查看权限

```bash
ls -l
```

示例：

```text
-rwxr-xr-- 1 root root 1234 Apr 21 script.sh
```

含义：

- 第一个字符表示类型：`-` 是文件，`d` 是目录。
- 后面九个字符分成三组：所有者、用户组、其他人。
- `r` 表示读，`w` 表示写，`x` 表示执行。

## 修改权限

```bash
chmod +x script.sh
chmod 644 file.txt
chmod 755 script.sh
```

常见权限：

- `644`: 普通文本文件常用，所有者可写，其他人只读。
- `755`: 脚本或目录常用，所有者可写，其他人可读和执行。
- `600`: 私钥文件常用，只有所有者可读写。

不建议随便使用：

```bash
chmod 777 file
chmod -R 777 folder
```

`777` 表示任何人都能读、写、执行，安全风险很高。

## 修改所有者

```bash
chown user file.txt
chown user:group file.txt
chown -R user:group folder/
```

`chown -R` 会递归修改整个目录。执行前一定确认目录路径正确。

## sudo 是什么

```bash
sudo command
```

`sudo` 表示用管理员权限执行命令。它很强大，也意味着更容易造成系统级影响。

建议：

- 不知道命令作用时，不要加 `sudo`。
- 修改 `/etc` 下配置前先备份。
- 执行远程脚本前先查看脚本内容。
