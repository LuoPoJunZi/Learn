# 1.1 Linux 是什么，先学什么

Linux 是一种常见的服务器操作系统。很多 VPS、云服务器、网站后端、自动化脚本都运行在 Linux 上。

新手学习 Linux，不需要一开始背所有命令。更重要的是先建立几个概念：

- 文件系统：Linux 里几乎一切都可以看作文件。
- 当前目录：你执行命令时所在的位置。
- 权限：谁能读、写、执行某个文件。
- 进程：正在运行的程序。
- 服务：长期在后台运行的程序，比如 SSH、Nginx、Docker。
- 包管理器：安装、更新、卸载软件的工具。

## 最小学习路线

建议按这个顺序学：

1. 会看自己在哪里：`pwd`
2. 会列出文件：`ls`
3. 会进入目录：`cd`
4. 会创建、复制、移动、删除文件：`mkdir`、`cp`、`mv`、`rm`
5. 会查看文件内容：`cat`、`less`、`head`、`tail`
6. 会编辑配置：`nano` 或 `vim`
7. 会看服务和日志：`systemctl`、`journalctl`
8. 会安全登录服务器：`ssh`

## 常见目录

```text
/             根目录
/home         普通用户目录
/root         root 用户目录
/etc          系统配置文件
/var/log      日志文件
/usr/bin      常用程序
/tmp          临时文件
```

新手最常接触的是：

- `/home/用户名`
- `/root`
- `/etc`
- `/var/log`

## 操作前的安全习惯

执行命令前先问自己：

- 我现在在哪个目录？
- 这个命令会不会删除或覆盖文件？
- 有没有备份？
- 命令里有没有 `rm -rf`、`dd`、`mkfs`、重装系统脚本？
- 远程脚本来源是否可信？

查看当前目录：

```bash
pwd
```

查看将要操作的文件：

```bash
ls -lah
```
