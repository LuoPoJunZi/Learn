# 3.2 SSH 安全加固与日志排查

SSH 是远程管理服务器最常用的入口，也是最容易被扫描和爆破的入口。公网 VPS 几乎一定会遇到陌生 IP 尝试登录。

## 查看 SSH 日志

Debian / Ubuntu 常用：

```bash
journalctl -u ssh
journalctl -u ssh -f
```

CentOS / RHEL 常用：

```bash
sudo tail -f /var/log/secure
```

如果日志里出现类似 `Invalid user`、`Failed password`，通常表示有人在尝试爆破登录。

## 常见日志含义

```text
Invalid user test from 1.2.3.4
```

表示有人尝试用不存在的用户名登录。

```text
Failed password for root from 1.2.3.4
```

表示有人尝试用密码登录失败。

```text
Server listening on 0.0.0.0 port 22
```

表示 SSH 正在监听 IPv4 的 22 端口。

## 安全加固建议

编辑配置文件：

```bash
sudo nano /etc/ssh/sshd_config
```

常见加固项：

```ini
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
```

含义：

- 禁止 root 直接登录。
- 禁止密码登录。
- 启用密钥登录。

如果你还没有确认密钥登录能成功，不要急着关闭密码登录，否则可能把自己锁在服务器外面。

## 限制允许登录的用户

```ini
AllowUsers your_username
```

只允许指定用户登录。替换成你自己的用户名。

## 修改后检查配置

修改 SSH 配置后，先检查语法：

```bash
sudo sshd -t
```

没有输出通常表示配置语法没有明显错误。

然后重启服务：

```bash
sudo systemctl restart ssh
```

如果系统服务名是 `sshd`：

```bash
sudo systemctl restart sshd
```

重要建议：重启 SSH 前，不要关闭当前 SSH 窗口。另开一个新窗口测试能否登录，确认没问题后再退出旧窗口。

## 防火墙限制来源 IP

如果你的 IP 比较固定，可以只允许特定 IP 访问 SSH：

```bash
sudo ufw allow from 192.168.1.100 to any port 22
```

实际使用时，把 `192.168.1.100` 换成你的真实可信 IP。

## 关于残留 sshd 进程

原始笔记中提到 `ssh.service: Found left-over process`。这通常表示服务重启或停止时有子进程没有被完全清理。

可以先查看状态：

```bash
sudo systemctl status ssh
ps aux | grep sshd
```

不建议新手直接 `killall sshd`，因为这可能断开当前远程连接。更稳妥的做法是先确认是否还有其他可用登录方式，再处理异常进程。
