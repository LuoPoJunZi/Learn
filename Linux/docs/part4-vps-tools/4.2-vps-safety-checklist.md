# 4.2 VPS 操作安全清单

VPS 上的命令往往影响整台服务器。尤其是远程脚本、DD 重装、网络调优、SSH 配置，执行前一定要确认风险。

## 执行脚本前

先问自己：

- 脚本来源可靠吗？
- 是否来自原作者仓库？
- 是否会重装系统、修改 SSH、修改防火墙？
- 是否会修改网络配置导致断连？
- 是否有备份？
- 是否有服务商控制台或 VNC 兜底？

## 不要盲目执行远程脚本

不建议直接执行：

```bash
curl -fsSL https://example.com/install.sh | bash
```

更稳妥的方式：

```bash
curl -fsSL https://example.com/install.sh -o install.sh
less install.sh
bash install.sh
```

或者：

```bash
wget https://example.com/install.sh -O install.sh
less install.sh
bash install.sh
```

先看脚本内容，再决定是否执行。

## 高风险操作

这些操作需要格外谨慎：

- DD 重装系统
- 修改 SSH 端口或关闭密码登录
- 修改防火墙规则
- 修改网络栈参数
- 执行 `rm -rf`
- 执行来源不明的一键脚本
- 安装非官方面板或破解面板

## SSH 配置修改前

修改 SSH 前建议：

1. 保持当前 SSH 会话不要关闭。
2. 确认服务商控制台可以登录。
3. 修改配置后先执行：

```bash
sudo sshd -t
```

4. 重启 SSH：

```bash
sudo systemctl restart ssh
```

5. 新开一个终端测试能否登录。

确认新连接成功后，再关闭旧连接。

## DD 重装前

DD 重装通常会清空系统盘。执行前确认：

- 重要数据已经备份。
- 服务器 IP、网关、密码或密钥已记录。
- 服务商允许自定义系统。
- 你知道默认账号密码是什么。
- 你能通过控制台救援。

## 脚本来源记录

建议在自己的笔记里记录：

```text
脚本名称：
来源链接：
执行时间：
服务器：
执行目的：
执行前备份位置：
执行结果：
```

这样以后排查问题时不会只剩一句“我好像跑过一个脚本”。
