# Linux VPS 基础安全清单

这份清单适合刚买 VPS、刚开始部署服务的新手。它不追求企业级安全，而是先帮你避开最常见、最危险的低级坑。

## 初始化时先做什么

建议顺序：

1. 记录系统版本和服务器用途。
2. 更新系统软件包。
3. 创建普通用户。
4. 配置 SSH 密钥。
5. 限制 root 或密码登录。
6. 配置防火墙或云安全组。
7. 记录开放端口和已安装服务。

查看系统版本：

```bash
cat /etc/os-release
```

## SSH 安全

优先使用 SSH 密钥登录。密码登录如果必须开启，也要使用强密码。

建议检查：

- SSH 端口是否明确。
- 是否已经配置密钥。
- 是否限制 root 直接登录。
- 是否保留备用登录方式。
- 修改 SSH 配置前是否有控制台或快照兜底。

修改 SSH 配置前，务必先读 [SSH 安全加固](docs/part3-network-ssh/3.2-ssh-security.md)。

## 防火墙和端口

只开放必要端口。

常见端口：

| 端口 | 用途 |
| :--- | :--- |
| 22 | SSH，可能已改成其他端口 |
| 80 | HTTP |
| 443 | HTTPS |

查看监听端口：

```bash
ss -tulnp
```

如果一个端口不知道是什么服务在用，不要随手放行到公网。

## 软件和脚本来源

安装软件前先确认来源：

- 官方仓库。
- 官方文档。
- 可信开源项目。
- 自己能读懂的一键脚本。

远程脚本尤其要谨慎：

```bash
bash <(curl -sL example.com/install.sh)
```

这种命令会把远程内容直接交给 shell 执行。运行前最好先打开脚本内容确认。

## 文件权限

不要用 `chmod 777` 解决所有问题。

更好的排查顺序：

1. 看当前用户：`whoami`
2. 看文件权限：`ls -lah`
3. 看服务运行用户。
4. 只给需要的最小权限。

权限越大，风险越大。

## 备份

至少备份：

- 网站配置。
- 数据库。
- 证书。
- 重要脚本。
- 环境变量说明。

备份不只是复制文件，还要确认能恢复。新手可以先写一份“恢复步骤”。

## 日志

出现异常时先看日志：

```bash
journalctl -xe
journalctl -u 服务名 -n 100
```

日志里常见关键词：

- `permission denied`
- `address already in use`
- `not found`
- `failed`
- `timeout`

## 高风险操作清单

执行前停一下：

- `rm -rf`
- `chmod -R`
- `chown -R`
- 修改 SSH 配置。
- 修改防火墙规则。
- 运行远程一键脚本。
- 删除日志、数据库、证书目录。
