# 3.1 什么是分支？

分支可以理解成项目的“平行版本”。你可以在一个分支上开发新功能，而不影响主分支里的稳定内容。

## 为什么需要分支

如果所有人都直接修改 `main`，很容易出现这些问题：

- 半成品代码被别人拉走
- 新功能还没测试就影响线上版本
- 多个人同时改动，难以追踪
- 出错后不好回退

分支的作用就是把不同工作隔离开。

## main 分支是什么

`main` 通常是项目主分支。团队一般会约定：

- `main` 保持稳定
- 不直接在 `main` 上开发新功能
- 新功能先在独立分支开发
- 通过 Pull Request 审查后再合并

## 常见分支命名

```text
feature/login-page
fix/button-style
docs/github-guide
refactor/user-service
```

建议分支名包含类型和主题。

常见类型：

- `feature`: 新功能
- `fix`: 修复问题
- `docs`: 文档修改
- `refactor`: 重构
- `chore`: 杂项维护

## 创建分支

开始前先更新主分支：

```bash
git switch main
git pull
```

创建并切换到新分支：

```bash
git switch -c feature/my-work
```

旧命令也可以：

```bash
git checkout -b feature/my-work
```

## 在分支上提交

```bash
git add .
git commit -m "feature: add my work"
```

推送新分支：

```bash
git push -u origin feature/my-work
```

## 合并分支

本地合并的基本命令是：

```bash
git switch main
git pull
git merge feature/my-work
```

但在团队中，更推荐通过 GitHub Pull Request 合并。这样别人可以先审查你的修改。

## 删除分支

功能合并后，可以删除本地分支：

```bash
git branch -d feature/my-work
```

删除远程分支：

```bash
git push origin --delete feature/my-work
```

## 一个推荐流程

```bash
git switch main
git pull
git switch -c feature/add-readme
git add .
git commit -m "docs: add readme section"
git push -u origin feature/add-readme
```

然后去 GitHub 创建 Pull Request。

## 新手记忆法

不要在主路上修车。先开一条临时路，把工作做好、检查完，再把结果并回主路。
