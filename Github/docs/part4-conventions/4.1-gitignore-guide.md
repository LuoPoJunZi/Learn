# 4.1 不该传的别传：.gitignore 配置规则

`.gitignore` 用来告诉 Git：哪些文件不需要进入版本管理。

它非常重要，因为有些文件上传到 GitHub 会带来麻烦，甚至造成安全风险。

## 哪些文件不该上传

常见不该上传的内容：

- 密码、Token、密钥
- `.env` 环境变量文件
- 日志文件
- 系统临时文件
- 编辑器个人配置
- 依赖目录
- 编译产物

## 一个通用示例

```gitignore
# 系统文件
.DS_Store
Thumbs.db

# 编辑器
.vscode/
.idea/

# 环境变量
.env
.env.*

# 日志
*.log

# 依赖
node_modules/

# 构建产物
dist/
build/
```

## 规则怎么写

忽略某个文件：

```gitignore
secret.txt
```

忽略某类文件：

```gitignore
*.log
```

忽略某个目录：

```gitignore
node_modules/
```

忽略目录下所有内容：

```gitignore
temp/*
```

例外规则：

```gitignore
*.log
!important.log
```

表示忽略所有 `.log` 文件，但保留 `important.log`。

## 已经提交过的文件怎么办

如果文件已经被 Git 跟踪，后来再写进 `.gitignore` 不会自动生效。

需要先从 Git 跟踪中移除，但保留本地文件：

```bash
git rm --cached 文件名
```

例如：

```bash
git rm --cached .env
```

然后提交：

```bash
git add .gitignore
git commit -m "chore: update gitignore"
```

## 密码已经上传怎么办

如果 `.env`、密钥、Token 已经上传到 GitHub：

1. 立即让旧密码或 Token 失效。
2. 重新生成新的密钥。
3. 从仓库中删除文件。
4. 必要时清理 Git 历史。

注意：只删除当前文件不代表历史记录里没有了。安全信息一旦泄露，应当默认已经不安全。

## 查找模板

GitHub 提供了很多 `.gitignore` 模板。创建仓库时可以选择语言，例如 Node、Python、Java。

也可以参考：

```text
https://github.com/github/gitignore
```

## 团队建议

- 项目一开始就添加 `.gitignore`
- 不要把个人编辑器偏好强加给团队
- 密钥和环境变量用 `.env.example` 提供示例
- 真正的 `.env` 不进入仓库
