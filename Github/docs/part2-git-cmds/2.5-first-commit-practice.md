# 2.5 第一次完整提交实战

这一章带你完整走一遍：修改文件、暂存、提交、推送到 GitHub。

如果你前面看懂了概念，但还不知道实际操作顺序，就从这里开始练。

## 练习目标

完成后你会知道：

- 怎么确认自己在正确的项目目录
- 怎么创建一个测试文件
- 怎么查看 Git 状态
- 怎么暂存修改
- 怎么提交到本地仓库
- 怎么推送到 GitHub
- 怎么确认 GitHub 上已经更新

## 开始前确认

你需要先准备好：

- 已安装 Git
- 已有 GitHub 账号
- 本地已经有一个 Git 仓库
- 仓库已经关联 GitHub 远程地址

如果还没有仓库，可以先看：

- [1.2 账号注册与安装 Git](../part1-basics/1.2-setup.md)
- [2.3 把本地已有文件夹推送到 GitHub](2.3-upload-local-project.md)

## 第一步：确认当前目录

打开 VS Code 终端，输入：

```bash
pwd
```

Windows PowerShell 也可以输入：

```powershell
Get-Location
```

确认当前目录是你的项目根目录。

项目根目录通常能看到：

```text
.git
README.md
```

如果不确定，输入：

```bash
git status
```

如果看到类似：

```text
On branch main
```

说明当前目录是 Git 仓库。

如果看到：

```text
not a git repository
```

说明你不在 Git 项目目录里，需要先 `cd` 到正确文件夹。

## 第二步：创建一个测试文件

新建文件：

```text
practice.md
```

写入：

```markdown
# Git 练习

这是我的第一次 Git 提交练习。
```

保存文件。

## 第三步：查看状态

输入：

```bash
git status
```

你可能会看到：

```text
Untracked files:
  practice.md
```

这表示 Git 发现了一个新文件，但它还没有被纳入暂存区。

## 第四步：暂存文件

暂存这个文件：

```bash
git add practice.md
```

再次查看状态：

```bash
git status
```

你会看到类似：

```text
Changes to be committed:
  new file: practice.md
```

这表示文件已经进入暂存区，准备提交。

## 第五步：提交到本地仓库

```bash
git commit -m "docs: add git practice note"
```

提交信息建议说明这次改了什么。

不要只写：

```text
update
```

更推荐：

```text
docs: add git practice note
```

## 第六步：推送到 GitHub

```bash
git push
```

如果是第一次推送当前分支，可能需要：

```bash
git push -u origin main
```

推送成功后，打开 GitHub 仓库页面，刷新，你应该能看到 `practice.md`。

## VS Code 图形界面怎么做

同样的流程可以用 VS Code 完成：

1. 新建并保存 `practice.md`
2. 打开左侧 Source Control
3. 点击 `practice.md` 旁边的 `+`
4. 在输入框写提交信息
5. 点击 `Commit`
6. 点击 `Sync Changes` 或 `Push`

对应关系：

| VS Code 操作 | Git 命令 |
| :--- | :--- |
| 点击 `+` | `git add` |
| Commit | `git commit` |
| Push / Sync | `git push` |

## 如果 push 失败怎么办

### 网络连接失败

常见报错：

```text
Failed to connect to github.com port 443
```

说明本地连接 GitHub 失败。可以先检查网络，再重试。

如果你需要配置 Git 代理，可以看：

- [2.4 用 VS Code 丝滑提交](2.4-vscode-workflow.md)

### 提示认证失败

可能原因：

- GitHub 账号没有登录
- HTTPS 凭据过期
- SSH 密钥没有配置好
- 当前账号没有仓库权限

如果你使用 SSH，可以回到：

- [1.3 配置 SSH 密钥](../part1-basics/1.3-ssh-keys.md)

### 提示 rejected

可能是 GitHub 上有别人更新过，你本地不是最新。

先执行：

```bash
git pull
```

如果出现冲突，看：

- [3.3 代码冲突怎么办？](../part3-teamwork/3.3-merge-conflict.md)

## 练习完成后可以删除测试文件吗

可以。

删除文件：

```bash
rm practice.md
```

Windows PowerShell：

```powershell
Remove-Item practice.md
```

然后提交删除：

```bash
git add practice.md
git commit -m "docs: remove git practice note"
git push
```

## 你应该记住的主线

```bash
git status
git add 文件名
git commit -m "说明这次改了什么"
git push
```

不知道下一步做什么时，先运行：

```bash
git status
```

它会告诉你现在有哪些文件被修改、哪些文件已暂存、下一步可以做什么。
