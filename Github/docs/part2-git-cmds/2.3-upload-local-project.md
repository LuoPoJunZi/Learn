# 2.3 如何把本地已有文件夹推送到 GitHub

这是新手最常见的场景：电脑里已经有一个项目文件夹，现在想把它上传到 GitHub。

## 总体思路

1. 在 GitHub 上创建一个空仓库。
2. 在本地项目文件夹里初始化 Git。
3. 提交本地文件。
4. 关联远程仓库。
5. 推送到 GitHub。

## 第一步：在 GitHub 创建空仓库

1. 登录 GitHub。
2. 点击右上角 `+`。
3. 选择 `New repository`。
4. 填写仓库名。
5. 先不要勾选 README、.gitignore、LICENSE。
6. 点击 `Create repository`。

这里建议先创建空仓库，因为本地已经有文件。如果 GitHub 同时生成了 README，本地推送时可能需要先处理历史差异。

## 第二步：进入本地项目文件夹

```bash
cd 路径/你的项目文件夹
```

确认当前目录正确：

```bash
pwd
```

Windows PowerShell 也可以用：

```powershell
Get-Location
```

## 第三步：初始化 Git

```bash
git init
```

执行后，当前文件夹会变成一个 Git 仓库。

## 第四步：添加 .gitignore

上传前建议先创建 `.gitignore`，避免把不该上传的文件传到 GitHub。

常见不该上传的内容：

- 密码文件
- `.env`
- 日志文件
- 临时文件
- 依赖目录，比如 `node_modules`
- 编译产物，比如 `dist`、`build`

示例：

```gitignore
.env
*.log
node_modules/
dist/
build/
```

## 第五步：提交本地文件

```bash
git add .
git commit -m "chore: initial commit"
```

这一步只是提交到本地，还没有上传到 GitHub。

## 第六步：关联远程仓库

在 GitHub 创建仓库后，页面会给你一个地址。HTTPS 形式类似：

```bash
git remote add origin https://github.com/用户名/仓库名.git
```

SSH 形式类似：

```bash
git remote add origin git@github.com:用户名/仓库名.git
```

检查是否关联成功：

```bash
git remote -v
```

## 第七步：推送到 GitHub

如果主分支叫 `main`：

```bash
git branch -M main
git push -u origin main
```

推送成功后，刷新 GitHub 仓库页面，就能看到本地文件。

## 完整命令模板

```bash
cd 你的项目文件夹
git init
git add .
git commit -m "chore: initial commit"
git branch -M main
git remote add origin https://github.com/用户名/仓库名.git
git push -u origin main
```

## 常见问题

### remote origin already exists

说明已经设置过远程地址。可以先查看：

```bash
git remote -v
```

如果地址错了，修改：

```bash
git remote set-url origin 新仓库地址
```

### 推送时提示 rejected

可能是远程仓库已经有提交。本地和远程历史不一致，需要先拉取：

```bash
git pull --rebase origin main
```

如果你不确定这样做是否安全，先备份项目文件夹，再继续。

### 不小心上传了密码

立即删除泄露的密钥或密码，并在对应平台重新生成。Git 历史里可能还保留旧内容，不能只删除文件就放心。
