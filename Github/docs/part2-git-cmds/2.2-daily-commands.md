# 2.2 日常指令备忘录

这一篇是 Git 常用命令速查表。刚开始不需要全部背下来，先记住最常用的一条主线：

```bash
git status
git add .
git commit -m "说明这次改了什么"
git push
```

## 下载仓库

从 GitHub 克隆一个仓库：

```bash
git clone https://github.com/用户名/仓库名.git
```

使用 SSH：

```bash
git clone git@github.com:用户名/仓库名.git
```

克隆后会在当前目录生成一个项目文件夹。

## 查看状态

```bash
git status
```

建议任何时候不知道下一步做什么，都先运行它。

## 查看修改内容

查看还没有暂存的修改：

```bash
git diff
```

查看已经暂存、准备提交的修改：

```bash
git diff --staged
```

## 暂存修改

暂存某个文件：

```bash
git add README.md
```

暂存当前目录下所有修改：

```bash
git add .
```

## 提交修改

```bash
git commit -m "docs: update README"
```

提交信息应该说明“为什么改、改了什么”，不要只写 `update`。

## 推送到 GitHub

```bash
git push
```

第一次推送新分支时，可能需要：

```bash
git push -u origin 分支名
```

## 拉取远程更新

```bash
git pull
```

在开始改代码前，建议先拉取一次，避免基于旧版本工作。

## 查看提交历史

```bash
git log
```

简洁版本：

```bash
git log --oneline
```

## 创建和切换分支

创建并切换到新分支：

```bash
git checkout -b feature/login
```

较新版本 Git 也可以用：

```bash
git switch -c feature/login
```

切换已有分支：

```bash
git switch main
```

## 查看分支

```bash
git branch
```

查看本地和远程所有分支：

```bash
git branch -a
```

## 撤销操作

撤销工作区中某个文件的修改：

```bash
git restore README.md
```

取消暂存：

```bash
git restore --staged README.md
```

注意：撤销命令可能让未保存的修改消失。执行前先确认自己真的不要这些内容。

## 常用组合

### 日常提交

```bash
git status
git add .
git commit -m "docs: add github setup guide"
git push
```

### 开始新功能

```bash
git pull
git switch -c feature/new-page
```

### 提交前自查

```bash
git status
git diff --staged
```
