# 2.1 工作区、暂存区、本地仓库

学习 Git 命令前，先理解三个区域：

- 工作区：你正在编辑的项目文件夹。
- 暂存区：准备被提交的一批修改。
- 本地仓库：已经正式保存下来的提交历史。

很多 Git 命令看起来难，其实只是文件在这三个区域之间移动。

## 工作区

工作区就是你能看到、能编辑的文件夹。

比如你用 VS Code 打开一个项目，修改了 `README.md`，这时修改只存在于工作区。

查看工作区状态：

```bash
git status
```

如果文件被修改但还没暂存，Git 会提示它是 modified。

## 暂存区

暂存区可以理解为“本次提交的购物车”。

你修改了很多文件，但不一定想一次性全部提交。可以只把想提交的文件放进暂存区。

把文件加入暂存区：

```bash
git add README.md
```

把所有修改加入暂存区：

```bash
git add .
```

`git add` 不等于上传到 GitHub，它只是告诉 Git：这些修改准备进入下一次提交。

## 本地仓库

本地仓库保存正式的提交记录。

提交暂存区内容：

```bash
git commit -m "docs: update README"
```

提交后，你的电脑里就有了一条新的历史记录。即使此时没有网络，也可以正常提交。

## 远程仓库

远程仓库通常在 GitHub 上。你本地提交后，还需要推送到远程，别人才能看到。

```bash
git push
```

从远程拉取别人最新的修改：

```bash
git pull
```

## 文件流转地图

```text
工作区 --git add--> 暂存区 --git commit--> 本地仓库 --git push--> GitHub
GitHub --git pull--> 本地仓库和工作区
```

## 一个完整例子

```bash
git status
git add README.md
git commit -m "docs: improve README"
git push
```

解释：

1. `git status` 查看现在有哪些修改。
2. `git add README.md` 把 README 的修改放入暂存区。
3. `git commit` 在本地保存一次修改记录。
4. `git push` 上传到 GitHub。

## 新手最容易误解的地方

### git add 不是上传

`git add` 只是暂存，不会上传到 GitHub。

### git commit 也不是上传

`git commit` 只是保存到本地仓库。没有执行 `git push`，GitHub 上看不到。

### git push 才是上传

`git push` 会把本地提交发送到远程仓库。

## 建议养成的习惯

每次操作前后都运行：

```bash
git status
```

它是新手最好的导航仪。看懂 `git status`，你就不容易迷路。
