# 1.1 Git 与 GitHub 到底有什么区别？

很多新手第一次听到 GitHub，会把 Git 和 GitHub 混在一起。它们确实经常一起出现，但不是同一个东西。

一句话区分：

- Git 是一个版本管理工具。
- GitHub 是一个托管 Git 仓库的网站和协作平台。

## Git 是什么

Git 可以帮你记录文件的每一次重要修改。

你可以把 Git 想成一个“高级存档系统”。写文档、写代码、做项目时，你可能遇到这些情况：

- 改错了，想回到昨天的版本
- 想知道某一行是谁改的
- 想同时尝试两个方案，最后选一个
- 多个人一起改同一个项目，不想互相覆盖

Git 就是为了解决这些问题而出现的。

## GitHub 是什么

GitHub 是一个网站。它可以把你的 Git 仓库放到网上，让别人也能访问、下载、参与修改。

你可以把 GitHub 想成：

- 项目的云端硬盘
- 程序员的协作平台
- 开源项目的展示主页
- 团队讨论需求、提交代码、审查代码的地方

没有 GitHub，你仍然可以在自己电脑上使用 Git。没有 Git，你也可以浏览 GitHub 上的网页。但真正的开发协作，通常会把它们一起使用。

## 仓库是什么

仓库的英文是 repository，常简称为 repo。你可以把它理解成“被 Git 管理的项目文件夹”。

一个仓库通常包含：

- 项目代码或文档
- 修改历史
- 分支
- 提交记录
- README 说明文档

在 GitHub 上，每个项目基本上就是一个仓库。

## 本地仓库和远程仓库

本地仓库在你的电脑里。远程仓库在 GitHub 上。

常见流程是：

1. 把 GitHub 上的仓库下载到本地。
2. 在本地修改文件。
3. 用 Git 保存修改记录。
4. 把本地记录推送回 GitHub。

这个过程常用到几个词：

- `clone`: 从远程复制一份仓库到本地
- `commit`: 在本地保存一次修改记录
- `push`: 把本地提交上传到远程
- `pull`: 把远程最新修改拉到本地

## 为什么不直接用网盘

网盘适合保存文件，但不擅长管理“变化过程”。

GitHub 和 Git 更适合项目协作，因为它们可以回答这些问题：

- 哪个文件在哪一天改过？
- 谁改了这一行？
- 这次修改解决了什么问题？
- 两个人同时改了同一个文件怎么办？
- 新功能能不能先审查，再合并进主项目？

这就是版本管理和普通文件同步的区别。

## 你现在只需要记住

- Git 是工具，GitHub 是网站。
- Git 管理版本，GitHub 托管仓库。
- 本地仓库在电脑上，远程仓库在 GitHub 上。
- 学 GitHub 的核心，是学会在本地修改、提交、推送，并和别人协作。
