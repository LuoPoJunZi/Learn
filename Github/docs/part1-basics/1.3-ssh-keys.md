# 1.3 配置 SSH 密钥

SSH 密钥可以让你的电脑和 GitHub 建立信任关系。配置成功后，你在推送代码时通常不需要反复输入账号密码。

## SSH 和 HTTPS 的区别

GitHub 仓库常见两种地址：

```text
HTTPS: https://github.com/user/repo.git
SSH:   git@github.com:user/repo.git
```

HTTPS 更容易理解，适合刚开始学习。SSH 配置稍微多一步，但长期使用更方便。

## 检查是否已有 SSH 密钥

打开终端，输入：

```bash
ls ~/.ssh
```

如果看到类似 `id_ed25519` 和 `id_ed25519.pub` 的文件，说明已经有密钥。

其中：

- 没有 `.pub` 后缀的是私钥，不能发给别人。
- 有 `.pub` 后缀的是公钥，可以添加到 GitHub。

## 生成新的 SSH 密钥

推荐使用 Ed25519：

```bash
ssh-keygen -t ed25519 -C "你的邮箱"
```

一路按回车即可。如果你愿意增加安全性，可以设置 passphrase；新手也可以先不设置。

生成后会得到：

```text
~/.ssh/id_ed25519
~/.ssh/id_ed25519.pub
```

## 复制公钥

Windows PowerShell：

```powershell
Get-Content ~/.ssh/id_ed25519.pub
```

macOS：

```bash
cat ~/.ssh/id_ed25519.pub
```

Linux：

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出的整行内容，从 `ssh-ed25519` 开始，一直到邮箱结束。

## 添加到 GitHub

1. 打开 GitHub。
2. 点击右上角头像。
3. 进入 `Settings`。
4. 找到 `SSH and GPG keys`。
5. 点击 `New SSH key`。
6. Title 随便写，例如 `My Laptop`。
7. Key 粘贴刚才复制的公钥。
8. 保存。

## 测试是否成功

输入：

```bash
ssh -T git@github.com
```

第一次连接时可能会询问是否信任 GitHub，输入：

```text
yes
```

如果看到类似下面的提示，说明成功：

```text
Hi username! You've successfully authenticated.
```

## 把仓库地址改成 SSH

如果你之前用 HTTPS 克隆了仓库，可以切换成 SSH：

```bash
git remote -v
git remote set-url origin git@github.com:用户名/仓库名.git
git remote -v
```

## 常见问题

### Permission denied

可能原因：

- 公钥没有添加到 GitHub
- 使用了错误的密钥
- 仓库地址写错
- 当前 GitHub 账号没有仓库权限

### 私钥能不能发给别人

不能。私钥相当于你电脑的身份凭证。只应该把 `.pub` 公钥添加到 GitHub。

### 一台电脑能配置多个 GitHub 账号吗

可以，但需要配置 `~/.ssh/config`。新手建议先只配置一个账号，熟悉后再学习多账号管理。
