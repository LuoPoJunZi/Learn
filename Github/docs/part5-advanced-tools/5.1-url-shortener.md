# 5.1 GitHub 脚本域名短链化与国内加速访问

有时我们会把脚本、安装命令或文档链接放在 GitHub 上。但 GitHub 链接可能很长，部分网络环境访问也不稳定。本篇介绍一些常见处理思路。

## 为什么需要短链

GitHub 原始链接可能长这样：

```text
https://raw.githubusercontent.com/user/repo/main/scripts/install.sh
```

如果要让别人复制执行，太长的链接容易换行、输错，也不方便记忆。

短链可以把它变成：

```text
https://example.com/install
```

## 常见方案

### 使用自己的域名跳转

你可以配置一个自己的短域名，把固定路径跳转到 GitHub 文件。

例如：

```text
https://go.example.com/install
```

跳转到：

```text
https://raw.githubusercontent.com/user/repo/main/install.sh
```

优点：

- 链接短
- 可控性强
- 后续迁移仓库时不用改用户入口

缺点：

- 需要域名
- 需要维护跳转服务

### 使用 Cloudflare Workers 做根域名脚本短链

如果你想把安装命令压缩到极简形式：

```bash
bash <(curl -sL sh.example.com)
```

可以使用 Cloudflare Workers。它的思路是：当用户访问 `https://sh.example.com/` 根路径时，Worker 去 GitHub 拉取真实脚本，然后把脚本内容以纯文本返回。

这种方式的好处是：

- 命令短，方便复制和传播
- 后续脚本地址变了，只需要改 Worker
- 可以统一加安全响应头和防缓存策略
- 可以拒绝非 GET 请求，减少异常请求

#### 第一步：创建 Worker

1. 登录 Cloudflare 控制台。
2. 进入 `Workers & Pages`。
3. 点击 `Create application`。
4. 选择 `Create Worker`。
5. 输入 Worker 名称，例如 `install-script`。
6. 部署后点击 `Edit code`。
7. 删除默认代码，替换为下面的版本。

```javascript
export default {
  async fetch(request) {
    if (request.method !== "GET") {
      return new Response("Method Not Allowed\n", { status: 405 });
    }

    const url = new URL(request.url);
    const commonHeaders = {
      "Content-Type": "text/plain; charset=utf-8",
      "Cache-Control": "no-store, no-cache, must-revalidate, proxy-revalidate",
      "X-Content-Type-Options": "nosniff"
    };

    if (url.pathname !== "/") {
      return new Response("Not Found\n", {
        status: 404,
        headers: commonHeaders
      });
    }

    const targetUrl = "https://raw.githubusercontent.com/用户名/仓库名/main/install.sh";

    try {
      const upstream = await fetch(targetUrl, {
        method: "GET",
        headers: {
          "User-Agent": "github-script-shortlink"
        },
        cf: {
          cacheTtl: 0,
          cacheEverything: false
        }
      });

      if (!upstream.ok || !upstream.body) {
        return new Response(`Upstream Error: HTTP ${upstream.status}\n`, {
          status: 502,
          headers: commonHeaders
        });
      }

      return new Response(upstream.body, {
        status: 200,
        headers: commonHeaders
      });
    } catch (error) {
      return new Response("Failed to connect to upstream server\n", {
        status: 502,
        headers: commonHeaders
      });
    }
  }
};
```

把 `targetUrl` 换成你的真实 GitHub 脚本地址，例如：

```text
https://raw.githubusercontent.com/your-name/your-repo/main/install.sh
```

#### 第二步：绑定自定义域名

1. 回到 Worker 详情页。
2. 打开 `Settings`。
3. 进入 `Triggers`。
4. 找到 `Custom Domains`。
5. 点击 `Add Custom Domain`。
6. 输入域名，例如 `sh.example.com`。
7. 等待 Cloudflare 自动配置 DNS 和 HTTPS 证书。

前提是你的主域名已经托管在 Cloudflare 上。通常等待几十秒到几分钟后就能生效。

#### 第三步：测试访问

浏览器访问：

```text
https://sh.example.com
```

如果页面直接显示 Bash 脚本内容，说明 Worker 已经生效。

终端测试：

```bash
curl -sL https://sh.example.com | head
```

确认内容没问题后，再使用安装命令：

```bash
bash <(curl -sL https://sh.example.com)
```

很多环境下也可以省略 `https://`：

```bash
bash <(curl -sL sh.example.com)
```

不过教程文档里建议保留 `https://`，让读者一眼看出脚本来源是 HTTPS 地址。

#### 这段 Worker 代码做了哪些保护

- 只允许 `GET` 请求，其他请求返回 `405`。
- 只允许访问根路径 `/`，其他路径返回 `404`。
- 使用 `Cache-Control: no-store`，避免用户拿到旧脚本。
- 使用 `X-Content-Type-Options: nosniff`，减少 MIME 嗅探风险。
- 上游出错时返回具体 HTTP 状态码，方便排查是 `404`、`429` 还是其他问题。
- 使用 Cloudflare `cf` 配置尽量绕过边缘缓存，确保脚本更新后能及时生效。

注意：这种方案只是缩短入口，不会让脚本天然更安全。真正的安全仍然取决于脚本内容、仓库可信度和用户是否愿意执行远程代码。

### 使用 GitHub Pages

如果只是放教程文档，可以使用 GitHub Pages，把仓库内容发布成网站。

常见地址：

```text
https://用户名.github.io/仓库名/
```

适合：

- 教程站点
- 项目文档
- 静态页面

不适合：

- 需要隐藏内容
- 需要后端服务
- 需要复杂权限控制

### 使用 CDN 或镜像加速

有些静态资源可以通过 CDN 分发，提高访问速度。

但需要注意：

- 不要把敏感文件放到公开 CDN
- 确认服务是否可靠
- 确认缓存更新机制
- 不要依赖不明来源的镜像执行脚本

## 安全提醒：不要盲目执行远程脚本

网上常见安装命令：

```bash
curl -fsSL https://example.com/install.sh | bash
```

这种方式很方便，但也有风险。它相当于把远程脚本直接交给本机执行。

更安全的做法：

```bash
curl -fsSL https://example.com/install.sh -o install.sh
less install.sh
bash install.sh
```

先下载、查看，再执行。

## 写给教程作者的建议

如果你要在文档里提供安装命令，建议同时提供：

- 短命令
- 原始 GitHub 链接
- 手动下载方式
- 校验说明
- 卸载方法

示例：

````markdown
## 快速安装

```bash
curl -fsSL https://go.example.com/install | bash
```

## 手动安装

1. 打开原始脚本地址：
   https://raw.githubusercontent.com/user/repo/main/install.sh
2. 保存为 install.sh。
3. 阅读脚本内容。
4. 执行：

```bash
bash install.sh
```
````

## 国内访问 GitHub 的建议

由于网络环境不同，访问 GitHub 的体验可能有差异。建议优先使用稳定、合规的网络环境。

可以考虑：

- 把教程内容发布到 GitHub Pages
- 同步一份文档到国内代码托管平台
- 为二进制文件提供多个下载源
- 在 README 中提供备用链接说明

不要在文档中推荐来路不明的代理、镜像或脚本加速地址。用户复制执行命令时，安全比速度更重要。

## 最佳实践

- 教程文档尽量使用短、稳定的入口链接
- 脚本链接保留原始 GitHub 地址，方便用户审查
- 不要隐藏脚本真实来源
- 关键安装脚本提供版本号或校验值
- 链接变更时优先维护跳转，不让旧文档失效
