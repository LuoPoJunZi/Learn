# Markdown 常见问题

Markdown 看起来简单，但新手常会在换行、链接、图片、表格和平台差异上踩坑。

## 为什么我写了换行，预览里却没有换行？

Markdown 中普通回车不一定会变成换行。更推荐用空行分段。

```markdown
这是第一段。

这是第二段。
```

如果只是想在同一段里强制换行，可以在行尾加两个空格，或使用 `<br>`。

## 标题为什么没有生效？

标题的 `#` 后面要有空格。

正确：

```markdown
## 标题
```

不推荐：

```markdown
##标题
```

一篇文档建议只有一个一级标题。

## 图片在本地能看，上传 GitHub 后看不到？

常见原因：

- 图片文件没有提交到仓库。
- 路径大小写不一致。
- 使用了本机绝对路径。
- Markdown 文件移动后，相对路径没有更新。

推荐使用相对路径：

```markdown
![截图](images/screenshot.png)
```

## 路径里有空格怎么办？

Markdown 链接遇到空格路径时，建议用尖括号包起来：

```markdown
[神经网络](<../Python/Neural Network/README.md>)
```

如果是新建目录，尽量使用英文小写和短横线，例如 `neural-network`。

## 表格为什么显示乱了？

Markdown 表格要求每一行列数尽量一致。

```markdown
| 名称 | 说明 |
| :--- | :--- |
| README | 项目说明 |
```

表格内容不要太长。长段文字更适合用列表或小标题。

## 代码块为什么把后面的内容都吞掉了？

通常是忘了写结束的三个反引号。

````markdown
```python
print("hello")
```
````

如果是在教程里展示代码块本身，可以用四个反引号包住。

## Mermaid 在所有平台都能显示吗？

不能。GitHub 支持 Mermaid，但不是所有博客、编辑器和主题都支持。

如果 Mermaid 图很重要，建议同时用文字解释流程，避免换平台后读者看不到图。

## README 要不要放很多徽章？

可以放，但不建议一开始就放太多。新手 README 最重要的是：

- 项目是什么。
- 怎么安装或打开。
- 怎么运行。
- 目录怎么找。
- 常见问题在哪里。

徽章属于锦上添花，不是核心内容。
