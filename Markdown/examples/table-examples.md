# Markdown 表格示例

表格适合整理短信息，例如入口、参数、命令、文件类型。不适合塞很长的段落。

## 目录入口表

```markdown
| 入口 | 说明 |
| :--- | :--- |
| README.md | 项目首页 |
| docs/ | 教程文档 |
| scripts/ | 维护脚本 |
```

## 参数说明表

```markdown
| 参数 | 是否必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `--input` | 是 | 无 | 输入文件路径 |
| `--output` | 否 | `result.txt` | 输出文件路径 |
```

## 对齐方式

```markdown
| 左对齐 | 居中 | 右对齐 |
| :--- | :---: | ---: |
| A | B | C |
```

## 写表格的提醒

- 表格不要太宽。
- 单元格里不要放很长段落。
- 路径或命令用反引号包起来。
- 如果移动端阅读很差，改成列表。
