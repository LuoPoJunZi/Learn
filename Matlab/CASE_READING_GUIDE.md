# Matlab 案例阅读指南

Matlab 目录里有大量算法和模型案例。新手不要从最大、最复杂的代码开始看，而要先学会“如何拆开一个案例”。

## 阅读一个案例前先看什么

建议先找到：

- README 或说明文件。
- 主运行脚本。
- 数据文件。
- 参数设置位置。
- 输出结果位置。
- 是否需要工具箱。

如果一上来就从所有 `.m` 文件里找逻辑，很容易迷路。

## 标准阅读顺序

```text
README
-> 主脚本
-> 数据读取
-> 参数设置
-> 核心算法函数
-> 结果输出和绘图
```

这个顺序适合多目标优化、神经网络、分类、回归、时间序列等大多数案例。

## 如何找到主脚本

常见线索：

- 文件名里有 `main`、`run`、`demo`、`test`。
- README 中写了运行入口。
- 脚本中包含数据读取、参数设置、调用算法、绘图输出。

如果找不到，先看 [示例运行索引](EXAMPLE_RUN_INDEX.md)。

## 数据文件怎么看

常见数据类型：

- `.mat`：Matlab 数据。
- `.xlsx`：Excel 数据。
- `.csv`：表格数据。
- `.jpg`、`.tif`：图像数据。

读取后先检查：

```matlab
size(X)
whos
```

不要在不理解数据维度的情况下直接改模型。

## 参数怎么看

算法案例常见参数：

- 种群规模。
- 迭代次数。
- 变量上下界。
- 目标函数数量。
- 网络层数或训练轮数。
- 学习率。

新手修改参数时，一次只改一个。这样结果变化才容易解释。

## 输出怎么看

输出可能包括：

- 命令行数值。
- 图像窗口。
- 保存的 `.mat` 文件。
- Excel 或图片结果。
- Pareto 前沿图。
- 分类准确率、回归误差、时间序列预测曲线。

看结果时先问：

- 结果是否符合任务目标？
- 图中横轴和纵轴是什么？
- 指标越大越好还是越小越好？
- 是否受到随机初始化影响？

## 多目标优化案例阅读重点

先理解：

- 决策变量是什么。
- 目标函数是什么。
- 约束条件是什么。
- Pareto 前沿表示什么。

再看算法：

- 初始化。
- 评价目标函数。
- 更新种群。
- 维护非支配解。
- 输出前沿。

可以先看 [多目标优化算法索引](<Multi-Objective Optimization/ALGORITHM_INDEX.md>)。

## 神经网络案例阅读重点

先判断任务类型：

- Classification：分类。
- Regression：回归。
- Time Series Prediction：时间序列预测。

再看：

- 输入特征。
- 输出标签或目标值。
- 训练集和测试集划分。
- 模型结构。
- 评价指标。

可以先看 [神经网络模型选择指南](<Neural Network/MODEL_SELECTION.md>)。

## 常见读代码误区

| 误区 | 建议 |
| :--- | :--- |
| 从第一个 `.m` 文件硬读 | 先找主脚本和 README。 |
| 一次改很多参数 | 一次只改一个关键参数。 |
| 只看结果图 | 同时看输入、指标和参数。 |
| 报错就重装 Matlab | 先检查路径、数据和工具箱。 |
| 忽略随机性 | 设置 `rng` 或多次运行取平均。 |

## 下一步

先用一个小案例练习这套阅读顺序。跑通默认版本后，把你理解的“输入、处理、输出”写成三行笔记，再尝试改参数。
