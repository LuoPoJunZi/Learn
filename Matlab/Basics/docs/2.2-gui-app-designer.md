# 2.2 GUI / App Designer 入门

Matlab 的 GUI 和 App Designer 适合把脚本封装成带按钮、输入框、图像显示区域的小工具。新手不要一开始追求复杂界面，先理解“控件触发回调函数”。

## 学习目标

学完后你应该能：

- 理解 GUI、App、控件、回调函数的关系。
- 知道 App Designer 适合什么场景。
- 把一个简单计算脚本改造成按钮触发。
- 明白界面代码和算法代码应该分开。

## GUI 的基本思路

GUI 的核心流程是：

```text
用户操作控件
-> 触发回调函数
-> 读取输入
-> 执行计算
-> 更新显示结果
```

例如点击按钮后，读取输入框中的数字，计算结果，再显示到标签或坐标轴里。

## 什么时候使用 App Designer

适合：

- 给算法做一个演示界面。
- 让不熟悉代码的人能点击运行。
- 做参数调节和结果展示。
- 做课程或实验演示工具。

不适合：

- 大型商业软件。
- 复杂多人协作项目。
- 需要跨平台 Web 部署的应用。

## 一个最小 App 的组成

通常包括：

- 输入框：输入参数。
- 按钮：触发计算。
- 坐标轴：显示图像或曲线。
- 标签：显示状态或结果。
- 回调函数：按钮点击后执行的代码。

## 设计建议

把算法写成普通函数，再在 App 里调用。

推荐结构：

```text
算法函数：负责计算
App 回调：负责读取输入、调用函数、显示结果
```

这样做的好处是：算法可以单独测试，界面也更容易维护。

## 常见问题

| 问题 | 建议 |
| :--- | :--- |
| 按钮点击没反应 | 检查回调函数是否绑定。 |
| 输入框内容不是数字 | 使用 `str2double` 转换并检查结果。 |
| 图画不到界面里 | 确认绘图目标是 App 中的坐标轴。 |
| App 代码越来越乱 | 把核心计算移到单独函数。 |

## 学习顺序

1. 先写一个普通脚本。
2. 把核心计算改成函数。
3. 在 App Designer 中创建按钮和输入框。
4. 在按钮回调里调用函数。
5. 把结果显示到标签或坐标轴。

## 下一步

学会最小 App 后，可以尝试把一个绘图脚本封装成 App：输入参数，点击按钮，更新曲线。这比一开始做复杂界面更稳。
