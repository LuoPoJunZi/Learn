# 2.5 并行计算入门

并行计算用于把可以独立执行的任务分给多个 worker 同时运行。它适合大量重复计算，但不是所有代码加上并行都会更快。

## 学习目标

学完后你应该能：

- 理解串行和并行的区别。
- 知道 `parfor` 适合什么情况。
- 明白并行计算有启动开销。
- 知道如何从普通 `for` 循环过渡到 `parfor`。

## 串行和并行

串行：

```text
任务 1 -> 任务 2 -> 任务 3 -> 任务 4
```

并行：

```text
worker 1: 任务 1
worker 2: 任务 2
worker 3: 任务 3
worker 4: 任务 4
```

如果每个任务互不依赖，并行才更容易发挥作用。

## `parfor` 基本形式

```matlab
result = zeros(1, 100);

parfor i = 1:100
    result(i) = i^2;
end
```

`parfor` 看起来像 `for`，但 Matlab 会尝试把循环分配给多个 worker。

## 适合并行的任务

适合：

- 大量独立仿真。
- 多组参数扫描。
- 多次随机实验。
- 图像批处理。
- 优化算法多次重复运行。

不适合：

- 每次循环都依赖上一次结果。
- 单次任务太小。
- 大量读写同一个文件。
- 代码瓶颈不在循环计算。

## 并行计算的成本

并行不是免费加速。它有：

- worker 启动开销。
- 数据传输开销。
- 内存占用增加。
- 调试复杂度增加。

如果任务很小，普通 `for` 可能更快。

## 学习建议

1. 先写普通 `for`。
2. 确认每次循环互不依赖。
3. 预分配输出变量。
4. 再改成 `parfor`。
5. 比较运行时间。

计时：

```matlab
tic
% your code
toc
```

## 下一步

如果你在多目标优化或神经网络案例中需要多次重复实验，可以先用普通循环跑通，再考虑并行化。不要一开始就为了“看起来高级”使用并行。
