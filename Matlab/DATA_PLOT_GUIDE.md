# Matlab 数据与绘图入门指南

很多 Matlab 案例的关键不在算法本身，而在数据是否读对、图是否解释清楚。这篇文档帮助新手先把数据和绘图基础打稳。

## 数据导入前先确认

先问自己：

- 数据文件在哪里？
- 文件格式是什么？
- 第一行是不是表头？
- 每一列代表什么？
- 缺失值怎么表示？
- 输出结果要保存到哪里？

这些问题不清楚时，算法跑出来也很难解释。

## 常见数据读取

读取 `.mat`：

```matlab
load data.mat
whos
```

读取 Excel：

```matlab
T = readtable("data.xlsx");
head(T)
```

读取数值矩阵：

```matlab
X = readmatrix("data.csv");
size(X)
```

读完后先检查大小和前几行，不要马上送进模型。

## 表格和矩阵怎么选

| 数据形态 | 推荐方式 |
| :--- | :--- |
| 有列名、混合类型 | `readtable` |
| 纯数值矩阵 | `readmatrix` |
| Matlab 保存变量 | `load` |

表格适合保留字段含义，矩阵适合数值计算。

## 绘图基本结构

一张可读的图至少包含：

```matlab
plot(x, y, "LineWidth", 1.5)
grid on
xlabel("x")
ylabel("y")
title("结果曲线")
legend("方法 A")
```

不要只画线。标题、坐标轴和图例是图的说明书。

## 多条曲线对比

```matlab
plot(x, y1, "LineWidth", 1.5)
hold on
plot(x, y2, "LineWidth", 1.5)
hold off
grid on
legend("方法 A", "方法 B")
```

对比图要说明每条线代表什么，否则读者只能猜。

## 保存图像

```matlab
saveas(gcf, "result.png")
```

保存前确认：

- 文件名是否有意义。
- 是否会覆盖旧结果。
- 图里是否包含标题、坐标轴和图例。

## 结果解释

看图时问：

- 横轴是什么？
- 纵轴是什么？
- 趋势是否符合预期？
- 异常点在哪里？
- 多次运行是否稳定？

算法结果不是“有图就行”。你要能解释图代表什么。

## 常见问题

| 问题 | 检查方向 |
| :--- | :--- |
| 文件找不到 | 当前工作目录、文件名、扩展名。 |
| 数据维度不对 | `size`、表头、缺失值。 |
| 图为空 | x/y 是否有数据，范围是否正确。 |
| 曲线很乱 | 数据是否排序，单位是否一致。 |
| 中文乱码 | 字体、编码和系统环境。 |
